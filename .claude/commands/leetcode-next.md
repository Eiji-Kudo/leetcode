# LeetCode 次のステップを進める

`docs/plan.md` の進捗を確認し、次の未完了ステップのテンプレートを生成してユーザーの学習を進める。
復習スケジュール（間隔反復）も管理し、適切なタイミングで復習を促す。

## 手順

### 0. 復習スケジュールの確認

`docs/progress.json` を読む。存在しない場合は `{"completions": {}}` で初期化してファイルを作成する。

本日の日付と各ステップの `nextReview` を比較し、**復習期日が今日以前のステップ**を抽出する。

#### progress.json の構造

```json
{
  "completions": {
    "1-1": {
      "completedAt": "2026-03-07",
      "reviews": [
        { "date": "2026-03-08", "passed": true },
        { "date": "2026-03-11", "passed": true }
      ],
      "intervalLevel": 2,
      "nextReview": "2026-03-18"
    }
  }
}
```

#### 間隔反復アルゴリズム（Spaced Repetition）

エビングハウスの忘却曲線に基づく復習間隔:

| intervalLevel | 次の復習まで | 意味 |
|---|---|---|
| 0 | 1日後 | 翌日の即時復習 |
| 1 | 3日後 | 短期記憶の定着 |
| 2 | 7日後 | 1週間後の確認 |
| 3 | 14日後 | 2週間後の確認 |
| 4 | 30日後 | 1ヶ月後の最終確認 |
| 5 | なし（卒業） | 長期記憶に定着済み |

**成功時**: `intervalLevel` を +1 し、対応する日数後を `nextReview` に設定する。Level 5 で卒業（`nextReview: null`）。

**失敗時**: `intervalLevel` を `max(0, current - 1)` に戻し、対応する日数後を `nextReview` に再設定する。

### 1. 進捗の確認

`docs/plan.md` を読み、チェックボックスの状態を確認する。

- `- [x]` → 完了済み
- `- [ ]` → 未完了

最初の未完了ステップ（`- [ ]`）を「次のステップ」として特定する。

完了済みステップの総数と全体数を数え、進捗サマリーを表示する。

**復習が必要なステップがある場合:**

```
📊 進捗: 5/82 完了（Step 1: 配列の基本操作 - 1-6 まで完了）
🔔 復習が必要: 1-3「for ループで配列を走査」(7日前に完了、復習期日: 3日前)
👉 まず復習から始めましょう → 1-3 の復習問題を出題します
```

復習が複数ある場合は**期日が最も古いもの**（最も遅延しているもの）を優先する。

**復習がない場合:**

```
📊 進捗: 5/82 完了（Step 1: 配列の基本操作 - 1-6 まで完了）
✅ 復習: 全て最新（次の復習: 1-3 → 3月15日）
👉 次のステップ: 1-7 reduce — 各要素を変換して新しい配列を返す
```

### 2. ステップの種類を判断

**復習期日のステップがある場合 → 間隔反復復習へ**（手順2-Rへ）

**復習がない場合**、次のステップのIDと説明から種類を判断する:

- **概念学習ステップ**（例: 1-1, 2-1, 5-1 など）→ **レッスンフェーズへ**（手順2-Lへ）
- **練習ステップ**（例: 1-9, 2-10 など `練習:` がついたもの）→ テンプレート生成へ（レッスン不要、直接問題を出す）
- **復習ステップ**（🔁マーク付き）→ 復習テンプレート生成へ（レッスン不要）
- **LeetCode問題の計画ステップ**（7-1a, 8-1a など `a` 付き）→ 問題解説 + アプローチ検討テンプレートへ
- **LeetCode問題の実装ステップ**（7-1b, 8-1b など `b` 付き）→ 実装テンプレートへ

#### 手順 2-L: レッスンフェーズ（概念学習ステップ）

概念学習ステップでは、**まずレッスン（授業）を行い、理解を確認してから練習問題に進む**。
ファイルの生成はまだ行わない。

**フェーズの判定方法:**
- 対応するステップのファイルがまだ存在しない → レッスンフェーズ
- ファイルが既に存在し TODO が残っている → ユーザーが練習中（レビュー待ち）
- ユーザーが「わかった」「把握した」「OK」「次へ」等と言った → 練習フェーズへ移行（手順3へ）

**レッスンの内容:**

1. **概念の説明**（チャットメッセージとして出力、ファイルは作らない）
   - その概念が何か、なぜ重要か（2〜3文）
   - 基本構文を提示

2. **具体例の提示**
   - シンプルなコード例を2〜3個、チャット上にコードブロックで示す
   - 各例に簡潔な説明を添える
   - 段階的に複雑さを上げる（基本 → 少し応用）

3. **理解確認**
   - 「ここまでの内容は大丈夫ですか？質問があれば聞いてください。OKなら練習問題に進みましょう！」と案内する
   - ユーザーが質問したら答える
   - ユーザーが理解を示したら（「わかった」「OK」「進もう」等）、手順3（ディレクトリの準備）→ 手順4（テンプレート生成）に進む

**レッスン出力のフォーマット例:**

```
## 📖 レッスン: for...of で配列を走査

### for...of とは？
for...of は配列の各要素を順番に取り出すシンプルな構文です。
従来の for ループと違い、インデックス変数を管理する必要がありません。

### 基本構文

\```javascript
for (const item of array) {
  // item に各要素が順番に入る
}
\```

### 例1: 配列の要素を出力

\```javascript
const fruits = ["apple", "banana", "cherry"];
for (const fruit of fruits) {
  console.log(fruit);
}
// => "apple"
// => "banana"
// => "cherry"
\```

### 例2: 合計を求める

\```javascript
const nums = [10, 20, 30];
let total = 0;
for (const n of nums) {
  total += n;
}
console.log(total); // => 60
\```

### for ループとの比較

\```javascript
// for ループ（インデックスが必要なとき）
for (let i = 0; i < arr.length; i++) {
  console.log(i, arr[i]);
}

// for...of（要素だけ必要なとき）
for (const item of arr) {
  console.log(item);
}

// インデックスも欲しい場合は entries()
for (const [i, item] of arr.entries()) {
  console.log(i, item);
}
\```

---
ここまでの内容は大丈夫ですか？質問があれば聞いてください。
OKなら練習問題に進みましょう！
```

#### 手順 2-R: 間隔反復復習の出題

復習対象ステップの元ファイルを読み、そのステップで学んだ概念を把握する。
**元の問題と同じ問題は出さず**、同じ概念を使う**新しい問題**を1問出題する。

ファイル名: `review-sr-{ステップID}.js`（例: `review-sr-1-3.js`）
配置先: 元のステップと同じディレクトリ

テンプレート:

```javascript
// =============================================
// 間隔反復復習: Step {ID}「{テーマ}」
// 完了日: {completedAt} / 復習回数: {reviews.length}回目
// =============================================

// 元のステップで学んだ概念:
// - {概念の要約}

// =============================================
// 復習問題: {新しい問題の説明}
// =============================================

function solution(/* 引数 */) {
  // TODO: ここに実装を書いてください
}

// --- テスト ---
console.assert(solution(/* テスト入力1 */) === /* 期待値1 */, "テスト1 失敗");
console.assert(solution(/* テスト入力2 */) === /* 期待値2 */, "テスト2 失敗");
console.assert(solution(/* テスト入力3 */) === /* 期待値3 */, "テスト3 失敗");

console.log("全テスト完了！エラーが表示されなければ正解です");
```

### 3. ディレクトリの準備

対応するステップのディレクトリが存在しない場合は作成する:
- Step 1 → `src/step01-array/`
- Step 2 → `src/step02-set-map/`
- Step 3 → `src/step03-string/`
- Step 4 → `src/step04-loops/`
- Step 5 → `src/step05-two-pointers/`
- Step 6 → `src/step06-hashmap/`
- Step 7 → `src/step07-easy-basic/`
- Step 8 → `src/step08-easy-advanced/`

`package.json` が存在しない場合は `npm init -y` を実行する。

### 4. テンプレートファイルの生成

plan.md のステップ情報（ID、テーマ、内容、ファイル名）に基づいて、以下の構成でテンプレートファイルを生成する。

**重要: 1ファイル = 1問題の原則**

1つのステップで複数の概念を扱う場合でも、**1ファイルにつき1つの問題（1つのTODO）** にする。
ユーザーが1問ずつ実装→実行→確認できるようにするため。

例: ステップで3つの概念を学ぶ場合 → 説明セクションで全概念を説明した後、問題は1つだけ出題する。
その1問で学んだ概念を総合的に使えるような問題にする。

**概念学習ステップのテンプレート（レッスン後）:**

レッスンフェーズで概念の説明は済んでいるので、テンプレートには説明を入れない。
問題とテストだけのシンプルな構成にする。

```javascript
// =============================================
// Step {ID}: {テーマ}
// =============================================

// =============================================
// 問題: {問題の説明}
// =============================================

function solution(/* 引数 */) {
  // TODO: ここに実装を書いてください
}

// --- テスト ---
console.assert(solution(/* テスト入力1 */) === /* 期待値1 */, "テスト1 失敗");
console.assert(solution(/* テスト入力2 */) === /* 期待値2 */, "テスト2 失敗");
console.assert(solution(/* テスト入力3 */) === /* 期待値3 */, "テスト3 失敗");

console.log("全テスト完了！エラーが表示されなければ正解です");
```

**練習ステップのテンプレート:**

練習ステップはレッスンなしで直接問題を出す。
ただし、問題に必要な最低限のコンテキスト（何を使って解くか等）は記載する。

```javascript
// =============================================
// Step {ID}: {テーマ}
// =============================================

// =============================================
// 問題: {問題の説明}
// =============================================

function solution(/* 引数 */) {
  // TODO: ここに実装を書いてください
}

// --- テスト ---
console.assert(solution(/* テスト入力1 */) === /* 期待値1 */, "テスト1 失敗");
console.assert(solution(/* テスト入力2 */) === /* 期待値2 */, "テスト2 失敗");
console.assert(solution(/* テスト入力3 */) === /* 期待値3 */, "テスト3 失敗");

console.log("全テスト完了！エラーが表示されなければ正解です");
```

**復習ステップのテンプレート:**
- 復習対象のステップ範囲から混合問題を出題
- 過去のステップで学んだパターンを組み合わせた問題にする
- 問題ごとにどのパターンを使うかのヒントは入れない（自分で判断させる）

**LeetCode 計画ステップ（a付き）のテンプレート:**
- LeetCode問題文の要約
- 入出力例
- 制約条件
- `// TODO: どのパターンが使えるか考えてみよう` セクション
- `// TODO: アプローチをコメントで書いてみよう` セクション

**LeetCode 実装ステップ（b付き）のテンプレート:**
- 関数シグネチャ
- テストケース（LeetCodeの例 + エッジケース）
- `// TODO: ここに実装を書いてください`

### 5. ユーザーへの案内

**レッスンフェーズの場合（手順2-Lで概念説明をした直後）:**
- ファイルは生成しない
- 「ここまでの内容は大丈夫ですか？質問があれば聞いてください。OKなら練習問題に進みましょう！」と案内する

**練習テンプレートを生成した場合:**
1. テンプレートファイルのパス
2. 実行コマンド: `node src/stepXX/{filename}`
3. 「TODO部分を実装したら教えてください。レビューします！」

### 6. レビュー（ユーザーが実装後に再度このスキルを呼んだ場合）

ユーザーが「できた」「レビューして」等と言った場合:

1. 該当ファイルを読む
2. `node` で実行してテストが通るか確認
3. テスト結果に応じてフィードバック:
   - 全テスト通過 → 褒める + 改善点があれば提案 + 別解の紹介
   - テスト失敗 → 失敗した旨と実行ログを表示する（ヒントはユーザーが求めるまで出さない）
4. 全テスト通過したら `plan.md` のチェックボックスを `- [x]` に更新
5. **progress.json を更新する**（→ 手順7へ）
6. 次のステップがあることを案内する: 「次のステップに進むには `/leetcode-next` と入力してください」

### 7. progress.json の更新

#### 通常ステップ完了時

テスト全通過後、`docs/progress.json` に以下を記録する:

```json
{
  "ステップID": {
    "completedAt": "YYYY-MM-DD（本日の日付）",
    "reviews": [],
    "intervalLevel": 0,
    "nextReview": "YYYY-MM-DD（明日の日付）"
  }
}
```

#### 間隔反復復習の完了時

**テスト全通過（成功）の場合:**

1. `reviews` 配列に `{ "date": "本日", "passed": true }` を追加
2. `intervalLevel` を +1
3. 新しい `intervalLevel` に応じて `nextReview` を計算:
   - Level 1 → 本日 + 3日
   - Level 2 → 本日 + 7日
   - Level 3 → 本日 + 14日
   - Level 4 → 本日 + 30日
   - Level 5 → `null`（卒業）
4. 復習用ファイル（`review-sr-*.js`）を削除する

**テスト失敗（不合格）の場合:**

1. `reviews` 配列に `{ "date": "本日", "passed": false }` を追加
2. `intervalLevel` を `max(0, current - 1)` に戻す
3. 新しい `intervalLevel` に応じて `nextReview` を再計算
4. ヒントを出して再挑戦を促す

#### plan.md の🔁復習ステップ完了時

通常のplan.md復習ステップ（🔁マーク付き）は、間隔反復とは別の仕組み。
チェックボックスの更新のみ行い、progress.json には記録しない。

### 8. 復習状況のサマリー表示

進捗表示時に、復習状況もあわせて表示する:

```
📊 進捗: 10/82 完了
🧠 復習状況:
  - 卒業済み: 1-1, 1-2, 1-3（3ステップ）
  - 次の復習: 1-4 → 3月12日, 1-5 → 3月15日
  - 復習期日超過: なし
👉 次のステップ: 1-11 ...
```
