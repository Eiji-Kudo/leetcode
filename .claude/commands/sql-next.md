# SQL 次のステップを進める

`docs/sql-plan.md` の進捗を確認し、次の未完了ステップのテンプレートを生成してユーザーのSQL学習を進める。
復習スケジュール（間隔反復）も管理し、適切なタイミングで復習を促す。

## 手順

### 0. 復習スケジュールの確認

`docs/sql-progress.json` を読む。存在しない場合は `{"completions": {}}` で初期化してファイルを作成する。

本日の日付と各ステップの `nextReview` を比較し、**復習期日が今日以前のステップ**を抽出する。

#### sql-progress.json の構造

```json
{
  "completions": {
    "1-1": {
      "completedAt": "2026-03-07",
      "reviews": [
        { "date": "2026-03-08", "passed": true },
        { "date": "2026-03-11", "passed": true }
      ],
      "intervalLevel": 2,
      "nextReview": "2026-03-18"
    }
  }
}
```

#### 間隔反復アルゴリズム（Spaced Repetition）

エビングハウスの忘却曲線に基づく復習間隔:

| intervalLevel | 次の復習まで | 意味 |
|---|---|---|
| 0 | 1日後 | 翌日の即時復習 |
| 1 | 3日後 | 短期記憶の定着 |
| 2 | 7日後 | 1週間後の確認 |
| 3 | 14日後 | 2週間後の確認 |
| 4 | 30日後 | 1ヶ月後の最終確認 |
| 5 | なし（卒業） | 長期記憶に定着済み |

**成功時**: `intervalLevel` を +1 し、対応する日数後を `nextReview` に設定する。Level 5 で卒業（`nextReview: null`）。

**失敗時**: `intervalLevel` を `max(0, current - 1)` に戻し、対応する日数後を `nextReview` に再設定する。

### 1. 進捗の確認

`docs/sql-plan.md` を読み、チェックボックスの状態を確認する。

- `- [x]` → 完了済み
- `- [ ]` → 未完了

最初の未完了ステップ（`- [ ]`）を「次のステップ」として特定する。

完了済みステップの総数と全体数を数え、進捗サマリーを表示する。

**復習が必要なステップがある場合:**

```
📊 SQL進捗: 5/70 完了（Step 1: SELECT基礎 - 1-5 まで完了）
🔔 復習が必要: 1-3「比較演算子」(7日前に完了、復習期日: 3日前)
👉 まず復習から始めましょう → 1-3 の復習問題を出題します
```

復習が複数ある場合は**期日が最も古いもの**（最も遅延しているもの）を優先する。

**復習がない場合:**

```
📊 SQL進捗: 5/70 完了（Step 1: SELECT基礎 - 1-5 まで完了）
✅ 復習: 全て最新（次の復習: 1-3 → 3月15日）
👉 次のステップ: 1-6 ORDER BY — 結果をソートする
```

### 2. ステップの種類を判断

**復習期日のステップがある場合 → 間隔反復復習へ**（手順2-Rへ）

**復習がない場合**、次のステップのIDと説明から種類を判断する:

- **概念学習ステップ**（例: 1-1, 2-1, 3-1 など）→ **レッスンフェーズへ**（手順2-Lへ）
- **練習ステップ**（例: 1-9, 2-9 など `練習:` がついたもの）→ テンプレート生成へ（レッスン不要、直接問題を出す）
- **復習ステップ**（🔁マーク付き）→ 復習テンプレート生成へ（レッスン不要）

#### 手順 2-L: レッスンフェーズ（概念学習ステップ）

概念学習ステップでは、**まずレッスン（授業）を行い、理解を確認してから練習問題に進む**。
ファイルの生成はまだ行わない。

**フェーズの判定方法:**
- 対応するステップのファイルがまだ存在しない → レッスンフェーズ
- ファイルが既に存在し TODO が残っている → ユーザーが練習中（レビュー待ち）
- ユーザーが「わかった」「把握した」「OK」「次へ」等と言った → 次のパートへ進むか、練習フェーズへ移行（手順3へ）

**レッスンの進め方: 少しずつ・対話的に**

一度に全部を説明するのではなく、**2〜3パートに分けて少しずつ教える**。
各パートの後に理解度を確認し、ユーザーの反応を見てから次に進む。

**パート構成（概念の内容に応じて2〜3パートに分ける）:**

**パート1: 概念の導入**
- その概念が何か、なぜ重要か（2〜3文で簡潔に）
- 基本構文を提示
- 最もシンプルなSQL例を1つ
- 「ここまで大丈夫ですか？」と確認

**パート2: 実践的な例**
- もう少し実用的なSQL例を1〜2個
- 段階的に複雑さを上げる
- 「ここまで大丈夫ですか？」と確認

**パート3（必要な場合のみ）: 比較・応用**
- 似た概念との比較や使い分け
- よくある間違いや注意点
- 「OKなら練習問題に進みましょう！」と案内

**各パートの分量の目安:**
- 1パートにつきSQLブロックは1〜2個まで
- 説明テキストは3〜5文まで
- 読むのに1分以内で済む量

**ユーザーの反応に応じた対応:**
- 質問があれば答える
- 「わかった」「OK」等 → 次のパートへ進む（最後のパートなら手順3へ）
- 「もっと詳しく」等 → 同じパートを掘り下げる

**パート1の出力フォーマット例:**

```
## 📖 レッスン: WHERE で条件絞り込み（1/3）

### WHERE とは？
WHERE句はSELECT文で取得する行を条件でフィルタリングします。
プログラミングの if 文のようなもので、条件に合う行だけが結果に含まれます。

### 基本構文

\```sql
SELECT 列名 FROM テーブル名 WHERE 条件;
\```

### 例: 年齢が30以上のユーザーを取得

\```sql
SELECT name, age
FROM users
WHERE age >= 30;
\```

| name    | age |
|---------|-----|
| Bob     | 30  |
| Charlie | 35  |

---
ここまで大丈夫ですか？
```

#### 手順 2-R: 間隔反復復習の出題

復習対象ステップの元ファイルを読み、そのステップで学んだ概念を把握する。
**元の問題と同じ問題は出さず**、同じ概念を使う**新しい問題**を1問出題する。

ファイル名: `review-sr-{ステップID}.js`（例: `review-sr-1-3.js`）
配置先: 元のステップと同じディレクトリ

テンプレート:

```javascript
const Database = require("better-sqlite3");

// =============================================
// 間隔反復復習: Step {ID}「{テーマ}」
// 完了日: {completedAt} / 復習回数: {reviews.length}回目
// =============================================

// 元のステップで学んだ概念:
// - {概念の要約}

// --- セットアップ（変更不要）---
const db = new Database(":memory:");

db.exec(`
  -- テーブル作成とテストデータ
`);

// =============================================
// 復習問題: {新しい問題の説明}
// =============================================

const query = `
-- TODO: ここにSQLクエリを書いてください
`;

// --- テスト（変更不要）---
const result = db.prepare(query).all();
console.assert(/* テスト1 */, "テスト1 失敗");
console.assert(/* テスト2 */, "テスト2 失敗");
console.assert(/* テスト3 */, "テスト3 失敗");

console.log("全テスト完了！エラーが表示されなければ正解です");
db.close();
```

### 3. ディレクトリの準備

対応するステップのディレクトリが存在しない場合は作成する:
- Step 1 → `src/sql-step01-select/`
- Step 2 → `src/sql-step02-aggregate/`
- Step 3 → `src/sql-step03-join/`
- Step 4 → `src/sql-step04-subquery/`
- Step 5 → `src/sql-step05-dml/`
- Step 6 → `src/sql-step06-window/`
- Step 7 → `src/sql-step07-practice-basic/`
- Step 8 → `src/sql-step08-practice-advanced/`

`better-sqlite3` がインストールされていない場合は `npm install better-sqlite3` を実行する。

### 4. テンプレートファイルの生成

sql-plan.md のステップ情報（ID、テーマ、内容、ファイル名）に基づいて、以下の構成でテンプレートファイルを生成する。

**重要: 1ファイル = 1問題の原則**

1つのステップで複数の概念を扱う場合でも、**1ファイルにつき1つの問題（1つのTODO）** にする。
ユーザーが1問ずつ実装→実行→確認できるようにするため。

**重要: テストデータの設計**

各問題では、学習する概念を適切にテストできるテストデータを用意する:
- テーブル構造はシンプルだが現実的なもの（users, products, orders 等）
- データは5〜10行程度（概念の理解に十分で、結果を目視確認しやすい量）
- エッジケースを含む（NULL値、重複、空の結果など、概念に応じて）

**概念学習ステップのテンプレート（レッスン後）:**

レッスンフェーズで概念の説明は済んでいるので、テンプレートには説明を入れない。
問題とテストだけのシンプルな構成にする。

```javascript
const Database = require("better-sqlite3");

// =============================================
// Step {ID}: {テーマ}
// =============================================

// --- セットアップ（変更不要）---
const db = new Database(":memory:");

db.exec(`
  CREATE TABLE テーブル名 (
    -- 列定義
  );

  INSERT INTO テーブル名 VALUES (...);
  -- テストデータ
`);

// =============================================
// 問題: {問題の説明}
// =============================================

const query = `
-- TODO: ここにSQLクエリを書いてください
`;

// --- テスト（変更不要）---
const result = db.prepare(query).all();
console.assert(result.length === 期待行数, "テスト1 失敗: N行返るべき");
console.assert(result[0].列名 === 期待値, "テスト2 失敗");
console.assert(result[1].列名 === 期待値, "テスト3 失敗");

console.log("全テスト完了！エラーが表示されなければ正解です");
db.close();
```

**練習ステップのテンプレート:**

練習ステップはレッスンなしで直接問題を出す。
ただし、問題に必要な最低限のコンテキストは記載する。

（テンプレート構造は概念学習ステップと同じ）

**復習ステップのテンプレート:**
- 復習対象のステップ範囲から混合問題を出題
- 過去のステップで学んだ概念を組み合わせた問題にする
- 問題ごとにどの概念を使うかのヒントは入れない（自分で判断させる）

**INSERT/UPDATE/DELETE ステップのテンプレート:**

DML（データ操作）系のステップでは、クエリの実行後にテーブルの状態を検証する:

```javascript
const Database = require("better-sqlite3");

// =============================================
// Step {ID}: {テーマ}
// =============================================

// --- セットアップ（変更不要）---
const db = new Database(":memory:");

db.exec(`
  CREATE TABLE テーブル名 (...);
  INSERT INTO テーブル名 VALUES (...);
`);

// =============================================
// 問題: {問題の説明}
// =============================================

const statement = `
-- TODO: ここにSQL文を書いてください
`;

// --- 実行 & テスト（変更不要）---
db.exec(statement);
const result = db.prepare("SELECT * FROM テーブル名").all();
console.assert(/* テスト1 */, "テスト1 失敗");
console.assert(/* テスト2 */, "テスト2 失敗");

console.log("全テスト完了！エラーが表示されなければ正解です");
db.close();
```

### 5. ユーザーへの案内

**レッスンフェーズの場合（手順2-Lで概念説明をした直後）:**
- ファイルは生成しない
- 「ここまでの内容は大丈夫ですか？質問があれば聞いてください。OKなら練習問題に進みましょう！」と案内する

**練習テンプレートを生成した場合:**
1. テンプレートファイルのパス
2. 実行コマンド: `node --watch src/sql-stepXX/{filename}`（ファイル保存のたびに自動で再実行される）
3. 「TODO部分にSQLクエリを書いたら教えてください。レビューします！」

### 6. レビュー（ユーザーが実装後に再度このスキルを呼んだ場合）

ユーザーが「できた」「レビューして」等と言った場合:

1. 該当ファイルを読む
2. `node` で実行してテストが通るか確認
3. テスト結果に応じてフィードバック:
   - **テスト失敗** → 失敗した旨と実行ログを表示する（ヒントはユーザーが求めるまで出さない）
   - **全テスト通過** → **ディスカッションフェーズ**（手順6-D）へ

#### 手順 6-D: ディスカッションフェーズ

テストが通った後、すぐに完了にせず、クエリについてディスカッションする。
以下の観点でユーザーのSQLクエリをレビューし、対話を促す:

1. **クエリの改善点**（あれば）
   - より読みやすい書き方、SQLの慣習的な書き方
   - パフォーマンスの観点（インデックスの活用、不要なサブクエリの排除など）
   - SQLフォーマットの統一（大文字/小文字、インデント）

2. **別解の紹介**
   - 同じ結果を別のアプローチで取得する方法を提示する
   - なぜそのアプローチが有効か、どんな場面で使い分けるか説明する
   - ただし、まだ学んでいない概念（後のステップで扱うもの）を使う場合は「これは後で学びます」と軽く触れる程度にする

3. **実務での活用**（任意）
   - 「実務ではこのクエリはどんな場面で使う？」のような問いかけ
   - データ量が増えた時の注意点

**フォーマット例:**

```
✅ 全テスト通過！

### クエリレビュー
（改善点やスタイルの指摘があればここに。なければ「シンプルで読みやすい良いクエリです」等）

### 別解
（別のアプローチを1つ紹介）

---
質問や気になることはありますか？
OKなら次のステップに進みましょう！
```

4. ユーザーが質問したら答える。ディスカッションを続ける。
5. ユーザーが「OK」「次へ」等と言ったら、ステップを完了にする:
   - `sql-plan.md` のチェックボックスを `- [x]` に更新
   - **sql-progress.json を更新する**（→ 手順7へ）
   - 「次のステップに進むには `/sql-next` と入力してください」と案内する

### 7. sql-progress.json の更新

#### 通常ステップ完了時

テスト全通過後、`docs/sql-progress.json` に以下を記録する:

```json
{
  "ステップID": {
    "completedAt": "YYYY-MM-DD（本日の日付）",
    "reviews": [],
    "intervalLevel": 0,
    "nextReview": "YYYY-MM-DD（明日の日付）"
  }
}
```

#### 間隔反復復習の完了時

**テスト全通過（成功）の場合:**

1. `reviews` 配列に `{ "date": "本日", "passed": true }` を追加
2. `intervalLevel` を +1
3. 新しい `intervalLevel` に応じて `nextReview` を計算:
   - Level 1 → 本日 + 3日
   - Level 2 → 本日 + 7日
   - Level 3 → 本日 + 14日
   - Level 4 → 本日 + 30日
   - Level 5 → `null`（卒業）
4. 復習用ファイル（`review-sr-*.js`）を削除する

**テスト失敗（不合格）の場合:**

1. `reviews` 配列に `{ "date": "本日", "passed": false }` を追加
2. `intervalLevel` を `max(0, current - 1)` に戻す
3. 新しい `intervalLevel` に応じて `nextReview` を再計算
4. ヒントを出して再挑戦を促す

#### sql-plan.md の🔁復習ステップ完了時

通常のplan.md復習ステップ（🔁マーク付き）は、間隔反復とは別の仕組み。
チェックボックスの更新のみ行い、sql-progress.json には記録しない。

### 8. 復習状況のサマリー表示

進捗表示時に、復習状況もあわせて表示する:

```
📊 SQL進捗: 10/70 完了
🧠 復習状況:
  - 卒業済み: 1-1, 1-2, 1-3（3ステップ）
  - 次の復習: 1-4 → 3月12日, 1-5 → 3月15日
  - 復習期日超過: なし
👉 次のステップ: 1-9 練習: SELECT + WHERE + ORDER BY 総合問題
```
